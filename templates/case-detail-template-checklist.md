# 玄星式详情页模板套用清单

这份清单用于每次新建相关详情页前后检查。建议新案例先复制 `case-detail-template.html`，再按本清单逐项替换。模板允许调整文字和图片，但不建议改变五章结构，因为当前玄星页的叙事节奏已经比较稳定。

## 一、复制前准备

新案例应先确定项目英文文件夹名，例如 `renegade-chair`、`lone-warrior-chair` 或其他短横线命名。所有素材统一放入 `images/works/项目英文名/`，这样模板路径只需要替换 `{{项目文件夹}}` 一个变量。

| 检查项 | 是否完成 | 说明 |
|---|---|---|
| 项目文件夹已创建 | □ | 路径为 `images/works/{{项目文件夹}}/`。 |
| Hero 主视觉已准备 | □ | 命名为 `detail-hero.jpg`。 |
| 过程图 01–03 已准备 | □ | 分别为 `step01-research.jpg`、`step02-params.jpg`、`step03-sketch.jpg`。 |
| 过程图 04 四张已准备 | □ | 使用 `step04-a-modeling.jpg`、`step04-b-material.jpg`、`step04-c-lighting.jpg`、`step04-d-render.jpg`。 |
| 过程图 05 四张已准备 | □ | 使用 `step05-a-assets.jpg`、`step05-b-composite.jpg`、`step05-c-lighting.jpg`、`step05-d-final.jpg`。 |
| 完整详情页长图已准备 | □ | 推荐 `{{项目文件夹}}-01.webp` 至 `{{项目文件夹}}-16.webp`。 |

## 二、HTML 必换字段

模板中所有 `{{字段名}}` 都是必须替换的占位符。替换完成后，页面源代码中不应再出现双大括号。

| 字段类型 | 必换内容 | 建议写法 |
|---|---|---|
| 基础信息 | 项目名称、项目短名、客户品牌、年份、项目周期 | 保持真实、简短、专业。 |
| 定位信息 | 项目类型、产品品类、项目角色、工具列表 | 与作品集卡片保持一致。 |
| 主题信息 | 核心主题词、核心视觉问题、核心卖点 | 尽量压缩成一句强表达。 |
| 第一章 | 竞品范围、竞品问题、差异点、引用来源 | 让用户看到这个设计是有判断的。 |
| 第二章 | 场景选择、姿态选择、色彩策略、信息密度 | 解释为什么画面这样做。 |
| 第三章 | 5 个过程步骤标题、说明和洞察 | 每个步骤都要对应图片内容。 |
| 第四章 | 屏数、每屏图片路径、首屏和末屏说明 | 默认展开，保持加高展示。 |
| 第五章 | 指标、结果、客户反馈或项目价值 | 有真实数据用数据，没有数据用交付价值。 |
| 翻页导航 | 上一个/下一个案例链接和名称 | 保持站内浏览连续。 |

## 三、上线前自检

替换完成后，必须先做一次桌面和移动端检查。重点不是追求每一处完全不同，而是确认页面逻辑、视觉节奏和完整成品展示成立。

| 自检项 | 通过标准 |
|---|---|
| 首屏是否有项目气质 | 打开页面 3 秒内能知道这是哪个产品、什么视觉主题。 |
| 第一章是否有明确问题 | 不是空泛介绍，而是有竞品、用户或业务问题。 |
| 第二章是否有统一主题词 | 后续画面都能回到这个主题词。 |
| 第三章图片和文案是否对应 | 文案说明的是图里能看到的内容，不写看不见的软件参数。 |
| 第四章是否默认展开 | 不需要点击即可看到完整详情页展示区。 |
| 第四章高度是否足够 | 桌面端约 1320px、移动端约 1080px，不再显得短。 |
| 长图是否连续 | 图片顺序正确、没有漏图、没有错误路径。 |
| 移动端是否可读 | 标题不溢出，长图区域可滚动，按钮不遮挡正文。 |
| 第五章是否可信 | 没有编造数据；没有数据时改写为项目价值。 |
| 源码是否无占位符 | 搜索 `{{` 和 `}}`，结果应为 0。 |

## 四、推荐新建流程

后续新建详情页时，建议按以下顺序执行。先替换素材路径和基础信息，再写第一、二章，最后写过程和结果，这样页面不会变成只换图的模板页。

| 步骤 | 操作 |
|---|---|
| 01 | 复制 `templates/case-detail-template.html` 为 `项目英文名-detail.html`。 |
| 02 | 在 `images/works/项目英文名/` 中放入所有图片素材。 |
| 03 | 全局替换 `{{项目文件夹}}`、`{{项目名称}}`、`{{项目短名}}` 等基础字段。 |
| 04 | 按 `xuanxing_case_template_guide.md` 完成五章文案。 |
| 05 | 检查完整详情页图片数量；若不是 16 屏，删除或增加对应 `<img>`。 |
| 06 | 搜索 `{{`，确保没有遗漏占位符。 |
| 07 | 本地预览桌面端和移动端，并重点检查第四章长图展示区。 |
| 08 | 更新 `portfolio.html` 和首页入口链接，再加版本参数规避缓存。 |

## 五、保留与可变部分

玄星式模板中，五章结构、首屏信息密度、过程滚动方式、第四章默认展开和加高长图容器建议保留。可变部分主要是背景图、主题词、案例文字、过程图数量、结果指标和上下篇链接。

| 建议保留 | 可以调整 |
|---|---|
| Hero + 五章 + 翻页导航 | 每章标题文案。 |
| 第一章问题意识 | 竞品类型和痛点来源。 |
| 第二章主题词统领 | 色彩和场景策略。 |
| 第三章 5 步过程 | 步骤名称和过程图内容。 |
| 第四章默认展开长图 | 图片数量和屏幕说明。 |
| 第五章结果收束 | 数据指标或项目价值表达。 |
