# 玄星式案例详情页模板规范

本文档用于把当前 **玄星 X2 人体工学椅详情页**沉淀为后续同类型案例的统一模板。后续电商详情页、3D 渲染详情页、产品视觉详情页，建议优先按此结构制作。这个模板的核心不是单纯复用版式，而是复用玄星页已经成立的叙事逻辑：**先讲问题，再讲视觉方向，再展示过程，最后给出完整成品与结果证明**。

> 模板原则：每个案例都要让访客看到“为什么这样做、怎么做出来、最终成品是什么、上线后有什么价值”。不要只堆大图，也不要只写过程文字。

## 一、页面总结构

玄星页建议固定为 **Hero + 五章正文 + 上下篇导航**。其中第四章“完整详情页”是重点展示区，必须默认展开，并保持足够高度，让访客一眼感受到作品完整度。

| 模块 | 模板名称 | 作用 | 后续替换重点 |
|---|---|---|---|
| 顶部导航 | 全站导航与面包屑 | 保持网站统一入口和当前位置感。 | 只替换当前案例名称。 |
| Hero 首屏 | 项目主视觉 | 迅速建立项目气质、客户、类型、周期和核心主题。 | 替换主视觉背景、项目名、主题词、项目元信息。 |
| 第一章 | 市场/用户问题 | 说明项目不是凭感觉做，而是从竞品、用户痛点或业务问题出发。 | 替换竞品洞察、用户差评、产品差异点。 |
| 第二章 | 视觉方向定义 | 用一个核心词定义整个视觉策略。 | 替换主题词、场景、人物/产品姿态、色彩策略、信息节奏。 |
| 第三章 | 从分析到执行过程 | 展示设计方法和专业能力，不只展示成品。 | 替换 5 个过程步骤和对应过程图。 |
| 第四章 | 完整详情页 | 展示最终 12–20 屏完整成品，是页面核心说服区。 | 替换完整详情页长图，默认展开，保持加高容器。 |
| 第五章 | 上线数据/项目反馈 | 用数据或反馈收束案例，让作品有商业结果。 | 替换转化率、停留率、阅读率、客户反馈；没有真实数据时改为项目价值总结。 |
| 翻页导航 | 上一个/下一个案例 | 保持作品浏览连续性。 | 替换相邻案例链接、名称和类型。 |

## 二、每一章的固定写法

第一章用于建立“问题意识”。推荐写法是：先说明项目起点，再说明调研方式，然后给出一句明确结论，最后用对比卡片把竞品共性和本项目差异点并列出来。这个章节不要写成空泛的“客户需要高端感”，而要尽量具体到用户痛点、平台竞争或视觉缺口。

| 第一章字段 | 写法建议 | 示例占位 |
|---|---|---|
| 章节标题 | 用一句问题式标题，引出矛盾。 | `竞品都在卷什么，用户真正在乎什么` |
| 项目起点 | 写清楚你先做了什么研究。 | `我先整理同价位竞品详情页，并逐条查看用户差评。` |
| 核心发现 | 必须有一个强结论。 | `用户抱怨的不是功能数量，而是坐感不够软。` |
| 引用/证据 | 可放用户评价、客户反馈或运营需求。 | `“坐垫太硬，坐久了大腿酸。”` |
| 对比卡片 | 左边放竞品共性，右边放本案差异。 | `工程感视觉` vs `柔软体感视觉` |

第二章用于定义“视觉方向”。推荐用一个主题词统领全页，例如玄星的“如坐云端”。这个词必须能同时解释画面、色彩、姿态和详情页节奏。后续所有图都应围绕这个词展开，不能前后风格割裂。

| 第二章字段 | 写法建议 | 示例占位 |
|---|---|---|
| 核心主题词 | 4–8 个字最好，便于记忆。 | `如坐云端`、`稳稳支撑`、`轻量通勤` |
| 场景选择 | 解释为什么选这个空间/背景。 | `云雾代表轻盈和柔软。` |
| 产品/人物姿态 | 解释姿态传递什么感受。 | `后仰舒展比端坐办公更能表达放松。` |
| 色彩策略 | 说明主色、强调色和品牌色关系。 | `冷色背景 + 橙色坐垫作为焦点。` |
| 信息密度 | 说明前后屏幕如何分配感性与理性内容。 | `前 8 屏情绪，后 8 屏功能参数。` |

第三章是模板的专业能力展示区。建议固定为 **5 个过程步骤**，分别对应调研、参数/卖点、结构草图或页面规划、3D/渲染、PS/合成/精修。每个步骤要有一张大过程图，滚动时先看图，再浮出说明文字。对于复杂步骤，可以在步骤后增加 4 宫格过程图，像玄星页的 C4D 和 PS 段落一样。

| 步骤 | 固定角色 | 可替换为 |
|---|---|---|
| 01 | 竞品截图分析 | 用户洞察、市场调研、平台页面拆解。 |
| 02 | 产品参数研究 | 卖点整理、功能拆解、材质分析。 |
| 03 | 草图与方向确认 | 详情页结构、视觉草图、脚本分镜、信息架构。 |
| 04 | C4D 建模渲染 | 建模、材质、灯光、场景搭建、渲染测试。 |
| 05 | PS 合成精修 | 素材拆分、图层合成、光影统一、最终落版。 |

第四章是最终成品展示区。后续所有同类页面都建议采用玄星页的处理方式：**默认展开、容器加高、内部滚动、16 屏左右连续展示**。这个区域不要再压缩为缩略图，因为完整长图本身就是电商详情页作品最重要的证明。

| 第四章规范 | 推荐值 |
|---|---|
| 展示状态 | 默认 `open`，进入页面即可看到成品。 |
| 桌面高度 | `max-height: min(1320px, 150vh)`。 |
| 移动端高度 | `max-height: min(1080px, 128vh)`。 |
| 图片数量 | 推荐 12–20 屏，玄星为 16 屏。 |
| 图片格式 | 推荐 `.webp`，宽度统一，命名连续。 |
| 图片命名 | `project-01.webp` 至 `project-16.webp`。 |
| 辅助说明 | 写明“从主视觉、功能卖点到参数收尾的完整购买说服链路”。 |

第五章用于收束。真实上线数据优先，如果没有公开数据，不要编造，可以改成“项目价值”或“交付结果”，例如“形成主视觉规范、完成 16 屏详情页、输出 C4D 场景资产、支持平台上线”。

## 三、素材命名规范

后续新案例建议在 `images/works/项目英文名/` 下建立独立文件夹。文件命名应保持连续、可读、可维护，这样模板替换时不容易出错。

| 素材类型 | 推荐命名 | 说明 |
|---|---|---|
| Hero 背景 | `detail-hero.jpg` | 首屏背景主视觉。 |
| 过程图 01 | `step01-research.jpg` | 调研/竞品分析。 |
| 过程图 02 | `step02-params.jpg` | 产品参数/卖点拆解。 |
| 过程图 03 | `step03-sketch.jpg` | 结构规划/草图/分镜。 |
| 过程图 04 | `step04-a-modeling.jpg` 等 | 建模、材质、灯光、渲染。 |
| 过程图 05 | `step05-a-assets.jpg` 等 | 素材、合成、光影、最终图。 |
| 完整详情页 | `project-01.webp` 至 `project-16.webp` | 最终成品长图，数量可按项目调整。 |

## 四、文案套用公式

后续每个案例可以按下面公式写。建议先把这 8 个问题填完，再开始改页面，否则容易变成单纯替换图片。

| 问题 | 填写内容 |
|---|---|
| 这个项目的客户是谁？ | `客户/品牌名称` |
| 产品是什么？ | `产品品类 + 核心卖点` |
| 用户真正关心什么？ | `痛点、差评、购买顾虑或平台竞争点` |
| 竞品都在怎么做？ | `常见画面套路和信息表达方式` |
| 本案要解决什么视觉缺口？ | `一个明确差异点` |
| 用哪个主题词统领视觉？ | `4–8 字主题词` |
| 详情页分几屏？ | `12–20 屏，最好说明前后段逻辑` |
| 最终结果如何证明有效？ | `数据、反馈、交付成果或项目价值` |

## 五、后续执行建议

后续每做一个相关详情页，建议先复制 `case-detail-template.html`，再按本文档替换项目变量、图片路径、五章文案和上下篇链接。若项目也是人体工学椅、电商详情页、3D 渲染类型，可以直接沿用玄星的章节顺序；如果是其他品类，也只需要替换“痛点、主题词、过程图、完整长图”，不建议改变五章结构。

这套模板最重要的价值，是让后续作品不再只是“图片展示页”，而是统一变成 **有判断、有过程、有成品、有结果的作品集案例页**。
